PIPEDA / data-safety design (Phase 3)
ConcernMitigation
Risk-score row tied to a real studentSHA-256(studentId + jwt-secret-salt). Insurer never sees the salt; the platform retains it. Single-source breach can't link hash → student.
Risk-score endpoint leaks individual dataOWNER-only in V1. B2B API key gate is TD before insurer enablement.
Aggregate endpoint leaks segmentsCounts-by-tier only. No school-level, jurisdiction-level, or time-bucket detail in V1.
Telemetry contains identifying infoOwner-curated payload — owner decides what's shipped. Schema is opaque JSONB so no required-fields contract leaks identity.
Telemetry dataset endpointCounts-only summary. Row-level export is TD with B2B API key.
