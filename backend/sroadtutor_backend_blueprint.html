What's actually built — backend modules done
Auth — signup & login
Email + password signup. Google login. Roles: Owner / Instructor / Student / Parent. Picks role at signup.
Done
JWT + refresh tokens
15-min access JWTs. 30-day refresh tokens, rotated on every use, with reuse-detection nuking all the user's tokens if a stolen one is replayed.
Done
Phone numbers (1..N)
Up to 5 phones per user / school / instructor / student. One marked primary. E.164 format.
Done
WhatsApp link generator
Generates wa.me click-to-chat links. Templates with placeholders OR free-text. Click-confirm verifies the recipient phone.
Done
Schools (NEW)
Owner creates one school. Tax IDs (GST/PST/HST). Plan tier. Soft-deactivate (no hard delete). One-school-per-owner enforced.
Done
Instructor profiles (NEW)
Self-register profile (vehicle, license, SGI cert, hourly rate, working hours JSON). Owner attaches them to school via M:N table. Soft delete.
Done
Student management (NEW)
Owner or instructor adds a student. One-shot creates the student User (dummy password, must rotate on first login) + Student row + optional Parent link.
Done
Parent linkage (NEW)
Adding a student can also link / create a parent by email. Existing parent → just link. New email → create PARENT user with dummy password.
Done
Standard error envelope
Every error returns { code, message, status, timestamp }. Stable codes for the SPA to handle. P0 prefix-bug fixed.
Done
Database migrations
9 Flyway migrations now (added V9 for is_active on schools + hourly_rate on instructors).
Done
Tenant scope checks
Every endpoint enforces "you can only see / edit your own school's data". 4-role × 4-resource matrix.
Done
Swagger / OpenAPI docs
Live API docs at /swagger-ui.html. Bearer-JWT auth built in.
Done
