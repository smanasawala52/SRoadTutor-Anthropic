What's actually built — backend modules done
Auth — signup & login
Email + password signup. Google login. Roles: Owner / Instructor / Student / Parent. Picks role at signup.
Done
JWT access tokens
15-minute access tokens. Sent in the Authorization: Bearer header. Stateless — no server session.
Done
Refresh tokens
30-day refresh tokens. Rotated on every use. If a stolen one is replayed, all the user's tokens get nuked.
Done
Password security
BCrypt strength 12 (~250ms per hash). Slow on purpose so brute force is expensive.
Done
Phone numbers (1..N)
Any user / school / instructor / student can have up to 5 phones. One is "primary". E.164 format.
Done
Phone scope rules
Owner sees own school. Instructor sees own. Student sees own. Parent sees linked student. Enforced on every call.
Done
WhatsApp link generator
Generates wa.me click-to-chat links. Pick a template OR write free-text. Logs every link for audit.
Done
Phone verification (click-confirm)
First time you click your own wa.me link, your phone is marked verified. No SMS OTP cost.
Done
WhatsApp templates
Reusable message bodies with {{name}} / {{lesson_time}} placeholders. Platform defaults + per-school overrides.
Done
Standard error envelope
Every error returns { code, message, status, timestamp }. Stable codes for the SPA to handle.
Done
Database migrations
8 Flyway migrations. Every schema change is a versioned SQL file. Never run "auto-DDL" in real environments.
Done
Swagger / OpenAPI docs
Live API docs at /swagger-ui.html. Bearer-JWT auth built in — paste a token, hit endpoints from the browser.
Done
